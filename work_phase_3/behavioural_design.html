Behavioural Design

Here I will display my work for this level of design.

This webpage highlights the principles of visceral design through intuitive layouts, a harmonious color scheme, and thoughtful typography.

Prototypes

Links

→Annotation

The vibrant color design for links effectively highlights them from texts satifsfying WCAG. Links exhibit intuitive and consistent behavior when focused or hovered over.

Navigation

→Annotation

To satisfy discoverability, the current page is always indicated with a bold font, helping users instantly recognize where they are. Hover states are clearly marked using background highlights as well. A persistent “Home” label is visible across both desktop and mobile views, allowing users to easily return to the starting point. The design adapts responsively: on mobile, navigation switches to a vertical layout with a hamburger menu, preserving clarity while maintaining access to all options. The layout order, typography, and spacing create a clear conceptual model—grouping related items and reflecting hierarchy—so users can predict navigation behavior and interact with confidence.

Cards

→Annotation

This card design enhances behavioural usability through immediate visual feedback and clear affordances. In its normal state, the card presents essential content in a clean layout. When hovered, multiple visual cues signal interactivity: the border highlights with a distinct color, the title ("Topic 1") becomes underlined, and the entire card gains subtle elevation. These changes act as signifiers that the card is clickable and lead to more detailed content. The consistent layout and hover behavior guide user expectations, reducing cognitive load and aligning with users’ mental models of interactive UI elements. Overall, the design helps users intuitively recognize clickable areas and reinforces navigational intent through responsive visual hierarchy.

Icons

→Annotation

Lables are displayed next to the icons indicating the function of each icon. The icons are designed to be intuitive and recognizable allowing users to quickly understand their purpose.

Button Styles

Annotation

Each button type uses consistent color-coding:

Green = Join actions (positive, go, submit)

Blue = Search (neutral/active utility)

Light = Toggle (Show/Hide)

Maroon = Disabled (inaccessible or completed actions)

All buttons have rounded rectangles and bold labels, providing clear affordances of clickability.

Hover and active states include shadow effects or darker borders, strengthening perceived interactivity (Feedback).

Form

This form allows users to register for a community by submitting required fields. The data is securely submitted to a dedicated API endpoint. Upon successful registration, users receive immediate on-screen confirmation. If fails, the corresponding error will be displayed as well.
